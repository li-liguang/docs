修改apt源​

版本：22.04 TLS

# deb-src https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy main restricted universe multiverse
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy-updates main restricted universe multiverse
# deb-src https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy-updates main restricted universe multiverse

mv sources.list sources.list.bak
vim sources.list

设置静态IP​

ifconfig查看电脑的网卡信息

找到物理网卡的名称，我这里是ens33

查看网关

输入route -n命令，打印路由表

修改网络配置文件

输入 sudo vim /etc/netplan/00-installer-config.yaml，编辑yaml文件，注意这里需要sudo权限。

配置生效

输入命令netplan apply使改动生效，然后reboot重启系统，就配置好了。

修改root密码​

sudo passwd

回车，输入新的密码，然后以下命令使用新设置的密码切换root用户验证：

su root

安装网络工具​

sudo apt install net-tools

安装远程ssh​

sudo apt install openssh-server

安装ssh工具后，发现无法使用root用户远程连接，这是因为系统默认禁止了root用户密码远程连接，解决方式：

sudo vim /etc/ssh/sshd_config

修改sshd_config文件，添加PermitRootLogin yes，然后保存退出，重启ssh服务

sudo service ssh restart

安装nodejs​

通过apt安装​

sudo apt-get install nodejs

升级nodejs

使用n模块。n模块是专门用来管理Node.js版本的，首先通过npm安装n模块，命令为sudo npm install -g n，然后使用n模块安装最新版本的Node.js，命令为sudo n latest。 n latest(升级node.js到最新版) or $ n stable（升级node.js到最新稳定版） n后面也可以跟随版本号比如：$ n v0.10.26 或者 $ n 0.10.26

通过nvm安装​

nvm是nodejs版本管理器，需要先安装nvm

安装nvm

wget -qO- https://raw.githubusercontent.com/creationix/nvm/v0.33.0/install.sh | bash

安装成功后，可能需要重新打开一个终端，nvm才会生效。 nvm相关命令（使用nvm --help查看）：

  • nvm -v：查看nvm的版本号
  • nvm install：安装nodejs；例如：nvm install v16.0.0
  • nvm list：列出已安装的所有nodejs版本
  • nvm use：切换指定版本的nodejs，例如：nvm use v16.0.0
  • nvm uninstall：卸载指定版本的nodejs，例如：nvm uninstall v16.0.0

安装开发工具​

安装JDK​

  • 安装方式一

通过apt包管理器安装

sudo apt-get update
# 安装
sudo apt install openjdk-8-jdk
# 检查安装版本
sudo java -version

可以使用sudo apt search jdk命令查找想要安装的jdk包。

  • 安装方式二

自己下载tar.gz安装包

这里选择的是Oracle的JDK，需要注册Oracle账号。

JDK 1.8

下载地址：https://www.oracle.com/java/technologies/downloads/#java8

下载压缩包jdk-8u351-linux-x64.tar.gz，不要下载rpm安装包。

sudo mkdir -p /dev-tools/jdk
# sudo mv /home/<user>/Downloads/jdk-8u351-linux-x64.tar.gz /dev-tools/jdk/
sudo mv /home/guang/下载/jdk-8u351-linux-x64.tar.gz /dev-tools/jdk/
cd /dev-tools/jdk
sudo tar -zxvf jdk-8u351-linux-x64.tar.gz
sudo rm -rf jdk-8u351-linux-x64.tar.gz

解压之后，修改环境变量：

sudo vim /etc/profile

添加如下配置：

export JAVA_HOME=/dev-tools/jdk/jdk1.8.0_351
# jdk8
export JRE_HOME=$JAVA_HOME/jre
export CLASSPATH=.:$JAVA_HOME/lib:$JRE_HOME/lib
export PATH=$JAVA_HOME/bin:$PATH

修改完成后，保存退出，执行source /etc/profile，使配置立即生效，然后执行java -version，打印java版本号，说明成功。

其他版本

JDK 11 下载地址：https://www.oracle.com/java/technologies/downloads/#java11

JDK 17 下载地址：https://www.oracle.com/java/technologies/downloads/#java17

同样下载压缩包，解压到/dev-tools/jdk/，只是不再配置环境变量。

问题记录

配置JDK环境变量后，重新打开终端，环境变量失效。 在网上找了一些解决方式，没有效果，最后重启系统解决！

安装IDEA​

下载地址：https://www.jetbrains.com/idea/download/#section=linux

下载链接：https://download.jetbrains.com/idea/ideaIU-2022.3.tar.gz

系统要求：

  • GNOME 或 KDE 桌面版
  • 最低 2 GB 可用 RAM，推荐 8 GB 系统总 RAM
  • 2.5 GB 硬盘空间，推荐 SSD
  • 最低屏幕分辨率 1024x768

安装说明：

  1. 使用下列命令将 idea-2022.3.tar.gz 文件解压到空目录：tar -xzf idea-2022.3.tar.gz

注：不得在现有实例上提取新实例。目标文件夹必须是空的。 1. 2. 从 bin 子目录运行 idea.sh

安装方式二：

可以使用命令行安装 IntelliJ IDEA。

sudo snap install intellij-idea-community --classic
# 或
sudo snap install intellij-idea-ultimate --classic
# 或
sudo snap install intellij-idea-educational --classic

激活IDEA：

使用ja-netfilter进行激活。激活网址：https://jetbra.in/s

  1. 访问上面的激活网址，任意打开一个可用链接
  2. 下载激活包jetbra.zip，按照包中的readme.txt文件中的配置说明进行配置
  3. 修改配置文件bin/idea64.exe.vmoptions，添加如下内容：
# 请修改为ja-netfilter.jar文件所在绝对路径
-javaagent:D:/Apps/JetBrains/jetbra-220801/ja-netfilter.jar=jetbrains
--add-opens=java.base/jdk.internal.org.objectweb.asm=ALL-UNNAMED
--add-opens=java.base/jdk.internal.org.objectweb.asm.tree=ALL-UNNAMED
  1. 获取激活码，点击激活网站上IDEA栏中的Copy to clipboard复制激活码
  2. 将复制的激活码添加到IDEA中即可

安装Docker​

官方安装文档：https://docs.docker.com/engine/install/ubuntu/

  1. 如果安装过docker，需要先卸载旧版本，这里没有安装过，直接跳过
  2. 添加apt仓库
  • 更新apt包索引，并安装包以允许apt使用HTTPS仓库
sudo apt-get update
sudo apt-get install ca-certificates curl gnupg lsb-release
  • 添加Docker官方GPG key
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
  • 设置仓库
echo \
"deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
$(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
  1. 安装Docker Engine
sudo apt-get update
# 安装最新版
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-compose-plugin

安装Nginx​

# 更新包索引
sudo apt update
# 安装nignx
sudo apt install nginx
# 确认是否安装成功并检查Nginx版本
nginx -v
# 启动
sudo systemctl start nginx
# 开机自启
sudo systemctl enable nginx

安装后，Nginx的配置文件所在目录：/etc/nginx

默认的80端口server在site-available下的default文件，这个文件在文件夹sites-enabled中有一个软连接，实际上nginx.conf中引入的配置是这个软连接。

软连接可通过以下命令创建：

sudo ln -s /etc/nginx/sites-available/my-server /etc/nginx/sites-enabled/
