Docker常用命令

基础命令​

# 启动
systemctl start docker
# 停止
systemctl stop docker
# 重启
systemctl restart docker
# 设置开机自启动
systemctl enable docker
# 取消开机自启动
systemctl disable docker
# 查看Docker状态
systemctl status docker
# 查看Docker概要信息
docker info
# 查看Docker版本
docker version
# 查看Docker帮助文档
docker --help

镜像操作​

搜索镜像​

docker search [镜像名称]
info

在镜像仓库上搜索指定的名称的镜像，默认是Docker Hub。

不过企业一般会使用Harbor搭建自己的镜像仓库。可以参考这里查看如何搭建Harbor仓库。

如果要在K8s中使用Harbor仓库，可以参考这里。

拉取镜像​

docker pull [镜像名称]:[版本号]
info

默认是拉取最新版本latest，例如：docker pull ngnix[:latest]。

默认是从Docker Hub上拉取，如果需要从Harbor上拉取，需要指定仓库地址，例如：docker pull harbor.test.com/infra/nginx:latest。

查看镜像​

docker images

删除镜像​

docker rmi [镜像ID|镜像名称]
info

如果要删除多个镜像，可以指定多个镜像ID，例如：docker rmi [镜像ID1] [镜像ID2]。

如果要删除所有镜像，可以指定-a参数，例如：docker rmi -a。

-f表示强制删除，例如：docker rmi -f [镜像ID]。

也可以使用查询镜像命令查看镜像ID，然后删除。

docker rmi -f $(docker images -aq)
info

-a 意思为显示全部, -q 意思为只显示ID。

构建镜像​

docker build -t [镜像名称]:[版本号] [Dockerfile所在目录]
info

例如：docker build -t mynginx:latest .。

-t 指定镜像名称和版本号，.表示当前目录，即Dockerfile所在目录。

保存[导出]镜像​

docker save -o [保存路径] [镜像名称]:[版本号]
info

例如：docker save -o /tmp/mynginx.tar mynginx:latest。

-o 指定保存路径，-o是--output的简写。

加载镜像​

docker load -i [镜像文件路径]
info

例如：docker load -i /tmp/mynginx.tar。

-i 指定镜像文件路径，-i是--input的简写。

给镜像打标签​

docker tag [镜像名称]:[版本号] [镜像名称]:[新版本号]
info

例如：docker tag mynginx:latest mynginx:v1.0。

给镜像打标签后，可以使用新的版本号进行操作。

也可以通过重新打标签的方式，更换镜像仓库地址。这样可以方便的将镜像推送到其他仓库。

docker tag [镜像名称]:[版本号] [仓库地址]/[镜像名称]:[版本号]

推送镜像​

docker push [镜像名称]:[版本号]
info

默认是推送到Docker Hub上，如果需要推送到Harbor上，需要指定仓库地址，例如：docker push harbor.test.com/infra/nginx:latest。

容器操作​

创建并运行容器​

docker run [选项] [镜像名称]:[版本号] [命令]
info

例如：docker run -d -p 80:80 --name mynginx nginx。

-d 表示后台运行，-p 表示端口映射，--name 表示容器名称，nginx 表示镜像名称。

如果需要指定容器的工作目录，可以使用-w参数，例如：docker run -d -p 80:80 --name mynginx -w /usr/share/nginx/html nginx。

常用选项：

  • -d：后台运行
  • -p：端口映射，格式为宿主机端口:容器端口
  • -v：挂载卷，格式为宿主机路径:容器路径
  • -e：设置环境变量，格式为KEY=VALUE
  • --name：容器名称
  • -w：容器内的工作目录

查看容器​

docker ps
info

默认只显示正在运行的容器，如果需要显示所有容器，可以添加-a参数，例如：docker ps -a。

停止/启动/重启容器​

docker stop [容器ID]
docker start [容器ID]
docker restart [容器ID]

删除容器​

docker rm [容器ID]
info

如果要删除多个容器，可以指定多个容器ID，例如：docker rm [容器ID1] [容器ID2]。

如果容器正在运行，可以使用-f强制删除。例如：docker rm -f [容器ID]。

进入容器​

docker exec -it [容器ID|容器名称] /bin/bash
info

例如：docker exec -it mynginx /bin/bash。

-it 表示以交互模式进入容器，/bin/bash 表示使用bash命令行。

查看容器日志​

docker logs [容器ID|容器名称]
info

默认只显示最后100行日志

-f参数实时显示日志，例如：docker logs -f mynginx。

--tail n：显示最后n行日志。

数据卷​

创建数据卷​

docker volume create [数据卷名称]
info

例如：docker volume create myvolume。

查看数据卷​

docker volume ls

查看数据卷详情​

docker volume inspect [数据卷名称]

删除数据卷​

docker volume rm [数据卷名称]
info

如果要删除多个数据卷，可以指定多个数据卷名称，例如：docker volume rm [数据卷名称1] [数据卷名称2]。

挂载数据卷​

docker run -v [数据卷名称]:[容器路径] [镜像名称]:[版本号]
info

例如：docker run -v myvolume:/usr/share/nginx/html nginx。

网络管理​

创建网络​

docker network create [网络名称]
info

例如：docker network create mynetwork。

通常是bridge类型的网络。

--driver或-d参数可以指定网络类型，例如：docker network create --driver bridge mynetwork。

查看网络​

docker network ls

查看网络详情​

docker network inspect [网络名称]

删除网络​

docker network rm [网络名称]

连接网络​

docker network connect [网络名称] [容器ID|容器名称]
info

例如：docker network connect mynetwork mynginx。

将容器连接到指定的网络。

断开网络​

docker network disconnect [网络名称] [容器ID|容器名称]
info

例如：docker network disconnect mynetwork mynginx。

将容器从指定的网络断开。

Docker Compose​

info

Docker Compose是一种定义和运行多容器Docker应用的工具，通过一个docker-compose.yml文件定义应用的服务和配置。

安装 Docker Compose​

sudo curl -L "https://github.com/docker/compose/releases/download/1.29.2/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

查看版本​

docker-compose --version

启动服务​

docker-compose up
info

根据docker-compose.yml文件中的配置启动所有服务。

默认创建并启动容器，如果容器已经存在，则重新创建并启动容器。

-d参数表示后台运行，例如：docker-compose up -d。

--build参数表示重新构建镜像，例如：docker-compose up --build。

--force-recreate参数表示强制重新创建容器，例如：docker-compose up --force-recreate。

--no-deps参数表示不启动依赖的服务，例如：docker-compose up --no-deps。

停止服务​

docker-compose down
info

停止并删除所有容器、网络、卷等资源。

构建或重建服务​

docker-compose build

查看服务状态​

docker-compose ps

查看服务日志​

docker-compose logs
info

默认只显示最后100行日志

-f参数实时显示日志，例如：docker-compose logs -f。

--tail n：显示最后n行日志。

查看服务配置​

docker-compose config

查看服务端口​

docker-compose port [服务名称] [容器端口]

查看服务环境变量​

docker-compose env [服务名称]

查看服务信息​

docker-compose inspect [服务名称]

查看服务资源使用情况​

docker-compose stats [服务名称]

常用清理命令​

info

Docker提供了一些简单的清理命令，用于删除不再使用的容器、镜像、卷和网络。

清理未使用的镜像​

docker image prune

清理未使用的容器​

docker container prune

清理未使用的网络​

docker network prune

清理未使用的卷​

docker volume prune

清理所有未使用的资源​

docker system prune
